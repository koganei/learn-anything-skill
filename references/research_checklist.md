# Research Checklist for Learning Topics

Use this checklist to ensure comprehensive research coverage when creating a learning path for any technical topic.

## Pre-Research Planning

### Define the Topic Scope

- [ ] **Topic name clearly defined:** What exactly are we teaching?
- [ ] **Boundaries established:** What's in scope vs. out of scope?
- [ ] **Target audience identified:** Beginner, intermediate, or mixed?
- [ ] **Context gathered:** Does the user have related experience?

### Example Scoping Questions

- "React hooks" - Do we cover all hooks or focus on common ones?
- "TypeScript" - Full language or specific features (types, generics, etc.)?
- "API design" - REST, GraphQL, both? Architecture principles?
- "Testing" - Unit, integration, E2E, or focused approach?

## Foundation Research (Beginner Level)

### Core Concepts

- [ ] **What is it?** Clear definition of the topic
- [ ] **Why does it exist?** What problem does it solve?
- [ ] **When was it introduced?** Historical context if relevant
- [ ] **Core terminology:** Key terms that must be understood
- [ ] **Basic syntax/patterns:** Fundamental building blocks

### Entry Point Content

- [ ] **Getting started guides:** Official quickstart resources
- [ ] **Simple examples:** "Hello World" equivalents
- [ ] **Common use cases:** Why beginners would use this
- [ ] **Prerequisites:** What must be known before starting
- [ ] **Setup requirements:** Tools, environment, dependencies

### Quality Checks

- [ ] Can explain the topic to someone with no background?
- [ ] Have 3-5 foundation concepts identified?
- [ ] Each concept has a simple, clear example?
- [ ] No assumptions about prior knowledge?

## Intermediate Research

### Building on Foundations

- [ ] **Common patterns:** How is this typically used in practice?
- [ ] **Composition:** How do basic concepts combine?
- [ ] **Real scenarios:** Practical applications beyond tutorials
- [ ] **State management:** How to handle complexity
- [ ] **Error handling:** Common failure modes and solutions

### Practical Application

- [ ] **Workflow integration:** How this fits into development process
- [ ] **Tools and libraries:** Ecosystem and supporting tools
- [ ] **Configuration:** Common setup and customization
- [ ] **Debugging:** How to troubleshoot issues
- [ ] **Testing approaches:** How to verify correctness

### Quality Checks

- [ ] Content assumes foundation knowledge?
- [ ] Have 3-5 intermediate concepts identified?
- [ ] Examples are realistic, not just demos?
- [ ] Shows how concepts work together?

## Advanced Research

### Complex Patterns

- [ ] **Architecture patterns:** Large-scale design approaches
- [ ] **Optimization:** Performance and efficiency considerations
- [ ] **Edge cases:** Handling complex or unusual scenarios
- [ ] **Scaling:** How to grow from small to large applications
- [ ] **Advanced features:** Power-user capabilities

### Production Considerations

- [ ] **Best practices:** Industry-standard approaches (2025-2026)
- [ ] **Anti-patterns:** What to avoid and why
- [ ] **Security:** Common vulnerabilities and protections
- [ ] **Performance:** Optimization techniques and profiling
- [ ] **Maintenance:** Long-term code health and refactoring

### Quality Checks

- [ ] Content assumes intermediate knowledge?
- [ ] Have 2-4 advanced concepts identified?
- [ ] Addresses real production concerns?
- [ ] Discusses trade-offs and alternatives?

## Use Case Research

### Scenario Identification

- [ ] **Day-to-day tasks:** Common operations developers perform
- [ ] **Integration scenarios:** Connecting with other systems
- [ ] **Migration/modernization:** Updating legacy approaches
- [ ] **Quality improvement:** Testing, refactoring, optimization
- [ ] **Specialized applications:** Domain-specific uses

### Scenario Details

For each use case:
- [ ] **Clear problem statement:** What are we trying to accomplish?
- [ ] **Step-by-step solution:** How to achieve the goal
- [ ] **Code examples:** Complete, runnable implementations
- [ ] **Common variations:** Different approaches to same goal
- [ ] **Gotchas:** Things that might go wrong

### Quality Checks

- [ ] Have 3-6 distinct use cases identified?
- [ ] Each use case is practical and realistic?
- [ ] Use cases cover different scenarios, not variations of one?
- [ ] Clear when to use each approach?

## Current State Research (2025-2026)

### Modern Best Practices

- [ ] **Latest version:** Current stable release and features
- [ ] **Recent changes:** What's new in past 1-2 years
- [ ] **Deprecated patterns:** Old approaches to avoid
- [ ] **Emerging trends:** Where the technology is heading
- [ ] **Community consensus:** What the community recommends now

### Tool Ecosystem

- [ ] **Standard tools:** What developers use daily
- [ ] **Alternative tools:** Other popular options
- [ ] **Integration tools:** Supporting services and libraries
- [ ] **Development tools:** IDE support, linters, formatters
- [ ] **Testing tools:** Framework-specific testing solutions

### Quality Checks

- [ ] Information is current (not from 2020)?
- [ ] Reflects 2025-2026 best practices?
- [ ] Mentions deprecated patterns when relevant?
- [ ] Acknowledges modern tooling?

## Resource Quality Assessment

### Source Credibility

- [ ] **Official documentation:** Primary source material
- [ ] **Authoritative blogs:** Well-known experts in the field
- [ ] **Recent articles:** Published within last 1-2 years
- [ ] **Community resources:** High-quality tutorials and guides
- [ ] **Video content:** When appropriate for learning style

### Information Verification

- [ ] **Cross-reference:** Multiple sources confirm key points
- [ ] **Version accuracy:** Information matches current versions
- [ ] **Code testing:** Examples actually work as described
- [ ] **Best practice validation:** Recommendations align with community
- [ ] **Conflict resolution:** Contradictions are investigated

### Quality Checks

- [ ] Primarily using official or authoritative sources?
- [ ] Information cross-checked across 3+ sources?
- [ ] No reliance on outdated tutorials?
- [ ] Conflicting information resolved?

## Content Organization Research

### Learning Path Structure

- [ ] **Logical progression:** Each topic builds on previous
- [ ] **Difficulty levels:** Clear foundation → intermediate → advanced
- [ ] **Topic count:** 12-20 topics for comprehensive coverage
- [ ] **Category balance:** Similar number of topics per category
- [ ] **Dependencies mapped:** Which topics require others

### Topic Naming

- [ ] **Clear and specific:** Names describe what's covered
- [ ] **Consistent level:** Similar scope across same difficulty
- [ ] **Action-oriented:** Often use verbs (Understanding, Building, Optimizing)
- [ ] **Searchable:** Users can find topics by name
- [ ] **Unique:** No two topics sound too similar

### Quality Checks

- [ ] Have 12-20 distinct topics identified?
- [ ] Topics organized into 3-5 categories?
- [ ] Clear progression from simple to complex?
- [ ] No major gaps in coverage?
- [ ] No significant overlap between topics?

## Completeness Verification

### Coverage Check

- [ ] **Foundations complete:** All prerequisite knowledge included
- [ ] **Common scenarios covered:** Daily tasks are addressed
- [ ] **Advanced features included:** Power-user capabilities present
- [ ] **Practical examples:** Real-world applications shown
- [ ] **Edge cases addressed:** Unusual scenarios mentioned

### Balance Assessment

- [ ] **Theory vs. practice:** Good mix of concepts and examples
- [ ] **Breadth vs. depth:** Comprehensive without being overwhelming
- [ ] **Beginner vs. advanced:** Appropriate content for each level
- [ ] **Explanation vs. code:** Balanced text and examples

### Quality Checks

- [ ] Could someone learn this topic end-to-end from this path?
- [ ] Are common questions answered?
- [ ] Are practical skills developed, not just theory?
- [ ] Is there a clear progression from novice to proficient?

## Special Considerations

### Topic-Specific Research

Different topics require different focus:

#### Programming Languages
- [ ] Syntax fundamentals
- [ ] Type system (if applicable)
- [ ] Standard library
- [ ] Common patterns and idioms
- [ ] Build tools and package management

#### Frameworks/Libraries
- [ ] Core concepts and philosophy
- [ ] Component/module structure
- [ ] State management approach
- [ ] Lifecycle and hooks
- [ ] Testing strategies

#### Architecture/Patterns
- [ ] Problem being solved
- [ ] When to use vs. alternatives
- [ ] Implementation approaches
- [ ] Trade-offs and considerations
- [ ] Real-world examples

#### Tools/DevOps
- [ ] Installation and setup
- [ ] Basic commands/operations
- [ ] Configuration options
- [ ] Integration with other tools
- [ ] Troubleshooting common issues

#### Practices/Methodologies
- [ ] Core principles
- [ ] Step-by-step workflows
- [ ] Benefits and drawbacks
- [ ] Adaptation to different contexts
- [ ] Measuring success

### Audience-Specific Adjustments

#### Complete Beginners
- [ ] More foundational content (topics 01-07)
- [ ] Simpler examples
- [ ] More explanation of "why"
- [ ] Fewer assumptions
- [ ] Slower progression

#### Intermediate Learners
- [ ] Less foundation, more application (topics 05-14)
- [ ] Real-world scenarios
- [ ] Integration patterns
- [ ] Best practices emphasis
- [ ] Common pitfalls

#### Advanced Learners
- [ ] Focus on topics 08-17
- [ ] Optimization and architecture
- [ ] Edge cases and complexity
- [ ] Trade-off discussions
- [ ] Production considerations

## Final Checklist

Before presenting the learning path to the user:

- [ ] 12-20 topics identified and organized
- [ ] Topics span foundations → intermediate → advanced
- [ ] Use case categories are logical and practical
- [ ] Each topic has clear description and purpose
- [ ] Dependencies between topics are clear
- [ ] Best practices reflect 2025-2026 standards
- [ ] Code examples are modern and accurate
- [ ] Sources are authoritative and recent
- [ ] Coverage is comprehensive without gaps
- [ ] Progression is logical and gradual
- [ ] Starting point recommendation is clear
- [ ] Research is documented for reference

## Time Management

### Recommended Research Duration

- **Simple topic (e.g., specific library):** 15-30 minutes
- **Medium topic (e.g., framework):** 30-60 minutes
- **Complex topic (e.g., architecture pattern):** 45-90 minutes
- **Comprehensive topic (e.g., language):** 60-120 minutes

### Efficient Research Strategy

1. **Start with official docs (10-20%):** Get authoritative foundation
2. **Survey recent tutorials (20-30%):** See practical applications
3. **Read best practice guides (20-30%):** Understand current approaches
4. **Review community discussions (10-20%):** Find common issues
5. **Organize and structure (20-30%):** Create learning path

### When to Stop Researching

Stop when you have:
- Clear understanding of foundations, intermediate, and advanced concepts
- 12-20 distinct topics identified and organized
- Confidence in creating comprehensive guides for each topic
- Modern best practices documented
- Practical examples for each difficulty level

Avoid:
- Researching every possible detail
- Getting stuck on minor variations
- Collecting exhaustive lists of resources
- Over-planning before starting to write
